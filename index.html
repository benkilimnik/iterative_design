Startup Chosen

Welcome to Mesh, a social networking site for remote companies! It allows employees to share the projects they are working on, explore other people's projects, and give each other feedback.

Wireframes

We took ideas from each of our sketches, and combined them into a single set of wireframes. We felt that the card-based dashboard clearly showed the user's current projects, while the tree structure was a good representation of all projects in the system. We also thought it was important for a project to have its own page and for there to be a page where users could submit feedback for another project.

Critique Feedback

- Text hierarchy (changed boldness, used one typeface)
- Dashboard icons (made notifications vs members clear)
- Project relationship (added parent project selection for the tree logic
- Unclear login page (added section describing Mesh)
- Unclear tree purpose (added text to indicate circles represent project hierarchy)
- Some lack of clarity in navigation (made buttons match the pages they go to, added breadcrumbs)

  1. Which task did you find the hardest? Why?
  2. Did the website's navigation match your Expectations?
  3. What would you change about this website if you could?

Analysis

Our Expectations

  • It should be pretty straightforward to create a new project and open a project to feedback
  • We were curious to find out if users can find other projects
    1. We expected some possible confusion around the explore page

User Behaviors & Mistakes

  • First user made no errors in tasks, did tasks pretty quickly but had trouble differentiating between own and others' projects
  • Second user tried to click the member circles from "My Project" page in order to go to other projects
  • All users had trouble finding the "Explore" page, they did not know that that would lead to seeing all possible projects
  • Users did not know what were "their projects" or "other people's projects". They did not instinctively click on the "i" in order to view the key

Potential Changes

  • Change header label from "explore" to "all projects"
  • Make key for projects tree onscreen
